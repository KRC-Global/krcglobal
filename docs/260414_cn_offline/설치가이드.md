# CN 분석 앱(CN_web) 내부망 추가 설치 가이드

작성일: 2026-04-14

---

## 1. 개요

본 가이드는 GBMS(해외사업관리시스템)가 설치된 내부망 서버에 CN 분석 앱(CN_web)을 추가로 설치하는 절차를 설명합니다.

CN_web은 별도 Flask 앱으로 포트 **5002**에서 실행되며, GBMS(포트 5001)와 독립적으로 동작합니다.

**분석 가능 범위 (내부망)**
- 한국 지역: 로컬 정밀토양도 + 피복도 파일 사용 → **정상 분석 가능**
- 해외 지역: SoilGrids WCS + ESA WorldCover API 필요 → **내부망에서 불가**

---

## 2. 전제 조건

설치 전 아래 사항이 완료되어 있어야 합니다.

| 항목 | 확인 |
|------|------|
| GBMS 기본 설치 완료 (preinstall2/ 방식) | ☐ |
| Python 3.11 설치됨 | ☐ |
| CN_web/ 폴더가 GBMS 루트와 같은 위치에 존재 | ☐ |
| 정밀토양도/ 폴더가 CN_web/cn_web/ 하위에 존재 | ☐ |
| 피복도/ 폴더가 CN_web/cn_web/ 하위에 존재 | ☐ |

**폴더 구조 예시**
```
C:\GBMS\
├── 해외사업관리시스템_외부망\   ← GBMS 루트
│   └── backend\
├── CN_web\
│   └── cn_web\
│       ├── app.py
│       ├── 정밀토양도\
│       └── 피복도\
└── migration\
    └── 260414\                  ← 본 설치 파일 위치
```

## 3. 설치 순서

### 3-1단계. 패키지 다운로드 (인터넷 연결 PC에서 실행)

인터넷이 연결된 PC에서 `01_패키지_다운로드.bat`를 실행합니다.

```
01_패키지_다운로드.bat 더블클릭
```

- `packages\` 폴더가 생성되고 필요한 wheel 파일이 다운로드됩니다.
- 완료 후 `packages\` 폴더 전체를 USB 등으로 내부망 서버로 복사합니다.

### 3-2단계. 패키지 복사

다운로드된 `packages\` 폴더를 내부망 서버의 `migration\260414\` 폴더 안에 복사합니다.

```
migration\
└── 260414\
    ├── packages\          ← 여기에 복사
    ├── requirements_cn.txt
    ├── 02_패키지_설치.bat
    └── ...
```

### 3-3단계. 패키지 설치 (내부망 서버에서 실행)

내부망 서버에서 `02_패키지_설치.bat`를 실행합니다.

```
02_패키지_설치.bat 더블클릭
```

- `packages\` 폴더의 wheel 파일을 오프라인으로 설치합니다.
- 오류 없이 완료되면 다음 단계로 진행합니다.

### 3-4단계. CN 앱 포트 설정 패치

`04_CN앱_포트설정_패치.py`를 실행하여 CN_web이 포트 5002를 사용하도록 설정합니다.

```
python 04_CN앱_포트설정_패치.py
```

- CN_web/cn_web/app.py 파일의 실행 포트가 5002로 변경됩니다.
- **한 번만 실행하면 됩니다.**

### 3-5단계. CN 앱 실행

`03_CN앱_실행.bat`를 실행합니다.

```
03_CN앱_실행.bat 더블클릭
```

- CN_web Flask 앱이 포트 5002로 실행됩니다.
- 창을 닫으면 서버가 종료됩니다. 운영 중에는 창을 유지하세요.

### 3-6단계. GBMS에서 CN 산정 메뉴 접속

브라우저에서 `http://localhost:5001`로 GBMS에 접속한 후 CN 산정 메뉴를 선택합니다.

GBMS는 내부적으로 `http://localhost:5002`로 CN_web에 요청을 보냅니다.

## 4. 포트 구성

| 앱 | 포트 | 비고 |
|----|------|------|
| GBMS 백엔드 (Flask) | 5001 | 기존 운영 포트 |
| CN_web (Flask) | 5002 | 본 설치로 추가 |
| GBMS 프론트엔드 (개발용) | 8000 | 개발 환경에서만 사용 |

## 5. 주의사항

1. **해외 지역 CN 분석 불가**: SoilGrids WCS, ESA WorldCover API는 외부 인터넷이 필요하므로 내부망에서는 동작하지 않습니다. 한국 지역 분석만 가능합니다.

2. **데이터 폴더 위치**: 정밀토양도/와 피복도/ 폴더는 반드시 `CN_web/cn_web/` 디렉토리 하위에 위치해야 합니다. 경로가 다르면 분석 시 오류가 발생합니다.

3. **포트 충돌 주의**: 포트 5002가 다른 프로그램에 의해 이미 사용 중이면 CN 앱이 실행되지 않습니다. 아래 문제해결 섹션을 참고하세요.

4. **패치 스크립트 중복 실행 금지**: `04_CN앱_포트설정_패치.py`는 최초 1회만 실행합니다. 중복 실행 시 `app.run()` 구문이 중복될 수 있습니다.

5. **서버 재시작 시**: 서버 재부팅 후 CN 앱은 자동으로 실행되지 않습니다. `03_CN앱_실행.bat`를 수동으로 다시 실행해야 합니다. 자동 실행이 필요하면 Windows 작업 스케줄러에 등록하세요.

## 6. 문제해결

### 문제 1: 포트 5002 이미 사용 중

**증상**: `Address already in use` 또는 `Only one usage of each socket address` 오류

**해결**:
```
netstat -ano | findstr :5002
```
결과에서 PID 확인 후:
```
taskkill /PID [PID번호] /F
```
이후 `03_CN앱_실행.bat` 재실행.

---

### 문제 2: 정밀토양도 폴더를 찾을 수 없음

**증상**: 분석 실행 시 `FileNotFoundError: 정밀토양도` 관련 오류

**해결**:
- `CN_web/cn_web/정밀토양도/` 폴더가 존재하는지 확인
- 폴더명 오탈자 확인 (공백, 대소문자 등)

---

### 문제 3: 패키지 설치 실패 (ModuleNotFoundError)

**증상**: `02_패키지_설치.bat` 실행 후 특정 모듈 import 오류

**해결**:
- `packages\` 폴더에 해당 패키지 wheel 파일(.whl)이 있는지 확인
- 인터넷 PC에서 `01_패키지_다운로드.bat`를 다시 실행하여 재다운로드
- Python 버전이 3.11인지 확인: `python --version`

---

### 문제 4: GBMS에서 CN 메뉴 접속 시 연결 오류

**증상**: CN 산정 메뉴 접속 시 "서버에 연결할 수 없습니다" 오류

**해결**:
- `03_CN앱_실행.bat` 창이 열려 있는지 확인 (닫혀 있으면 재실행)
- 브라우저에서 `http://localhost:5002` 직접 접속하여 CN_web 응답 확인
- Windows 방화벽에서 포트 5002 허용 여부 확인

---

## 7. 설치 완료 확인

아래 URL에서 모두 응답이 오면 정상 설치된 것입니다.

- GBMS: `http://localhost:5001` → 로그인 화면 표시
- CN_web: `http://localhost:5002` → CN_web 응답 확인
