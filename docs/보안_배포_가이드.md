## 2. 프로덕션 배포 절차

### 2.1 사전 준비

**필수 작업:**

1. **Flask-WTF 설치**
```bash
cd backend
pip install Flask-WTF==1.2.1
```

2. **시크릿 키 생성**
```bash
# Python을 사용하여 안전한 랜덤 키 생성
python -c "import secrets; print('SECRET_KEY:', secrets.token_urlsafe(32))"
python -c "import secrets; print('JWT_SECRET_KEY:', secrets.token_urlsafe(32))"
```

출력 예시:
```
SECRET_KEY: xK3mP9vR2nQ8wL5tF7jH4zB6cY1aS0dG
JWT_SECRET_KEY: pT8kM2vX5nL9qR3wF6hJ1zC4yB7aD0sG
```

⚠️ **주의**: 생성된 키는 안전한 곳에 보관하고, 절대 GitHub 등에 커밋하지 마세요!

### 2.2 환경 변수 설정 (Windows Server)

**방법 1: 시스템 환경 변수 설정 (권장)**

```cmd
# 관리자 권한으로 명령 프롬프트 실행
setx SECRET_KEY "생성된_시크릿_키" /M
setx JWT_SECRET_KEY "생성된_JWT_키" /M
setx FLASK_ENV "production" /M
```

**방법 2: .env 파일 사용**

```bash
# backend/.env.example을 backend/.env로 복사
cd backend
copy .env.example .env

# .env 파일을 편집하여 실제 키 입력
notepad .env
```

`.env` 파일 내용:
```env
FLASK_ENV=production
SECRET_KEY=xK3mP9vR2nQ8wL5tF7jH4zB6cY1aS0dG
JWT_SECRET_KEY=pT8kM2vX5nL9qR3wF6hJ1zC4yB7aD0sG
```

### 2.3 배포 전 검증

**체크리스트:**

```bash
# 1. 환경 변수 확인
echo %SECRET_KEY%
echo %JWT_SECRET_KEY%
echo %FLASK_ENV%

# 2. 의존성 설치 확인
pip list | findstr Flask-WTF

# 3. 데이터베이스 백업
copy backend\database\gbms.db backend\database\gbms_backup_%date:~0,4%%date:~5,2%%date:~8,2%.db
```

### 2.4 서버 재시작

```bash
# 기존 서버 종료 (Ctrl+C 또는 프로세스 종료)

# 새로운 서버 시작
cd backend
python app.py
```

또는 프로덕션 스크립트 사용:
```bash
start_prod.bat
```

### 2.5 배포 후 검증

**1. 서버 로그 확인**
```bash
type backend\server.log
```

**2. 보안 헤더 확인**
```bash
curl -I http://localhost:5001/api/health
```

예상 응답:
```
HTTP/1.1 200 OK
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
X-XSS-Protection: 1; mode=block
Content-Security-Policy: default-src 'self'; ...
Referrer-Policy: strict-origin-when-cross-origin
Permissions-Policy: geolocation=(), microphone=(), camera=()
```

**3. 비밀번호 정책 테스트**

로그인 후 비밀번호 변경 시도:
- ❌ "password1" → "비밀번호는 10자 이상이어야 합니다."
- ❌ "password123" → "대문자를 1개 이상 포함해야 합니다."
- ✅ "Password123!" → 성공

## 3. 기존 사용자 비밀번호 마이그레이션

### 3.1 관리자 공지

기존 사용자들에게 다음 내용을 공지하세요:

```
[중요] GBMS 보안 정책 강화 안내

보안 강화를 위해 비밀번호 정책이 변경되었습니다.

▶ 변경된 비밀번호 정책:
- 최소 10자 이상
- 대문자, 소문자, 숫자, 특수문자 각 1개 이상 포함

▶ 조치 사항:
- 다음 로그인 시 "비밀번호 변경" 메뉴에서 새로운 비밀번호로 변경해주시기 바랍니다.
- 기존 비밀번호가 정책에 맞지 않는 경우, 로그인은 가능하지만 비밀번호 변경이 필요합니다.

▶ 비밀번호 예시:
- ✅ GlobalKRC2026!
- ✅ Gbms@Admin2026
- ❌ admin123 (대문자, 특수문자 없음)

문의: 글로벌사업부 IT팀 (내선 6456)
```

### 3.2 관리자 비밀번호 리셋 스크립트

기존 `reset_admin_password.py`를 사용하여 관리자 계정 비밀번호를 강화된 정책에 맞게 변경:

```python

# 새 비밀번호 입력 시 강화된 정책 적용됨
```

### 5.2 HTTPS 적용 (내부망)

내부망에서도 중간자 공격(MITM) 방어를 위해 HTTPS 권장:

```bash
# 자체 서명 인증서 생성 (개발/테스트용)
openssl req -x509 -newkey rsa:4096 -nodes -out cert.pem -keyout key.pem -days 365

# Flask에서 HTTPS 실행
python app.py --cert=cert.pem --key=key.pem
```

### 5.3 파일 업로드 추가 검증


## 6. 문제 해결 (Troubleshooting)

### 문제 1: "프로덕션 환경에서는 SECRET_KEY 환경 변수가 필수입니다" 오류

**원인**: 환경 변수가 설정되지 않음

**해결**:
```cmd
setx SECRET_KEY "생성된_키" /M
setx JWT_SECRET_KEY "생성된_키" /M
# 명령 프롬프트 재시작 후 서버 재실행
```

### 문제 2: 기존 사용자 로그인 불가

**원인**: 기존 비밀번호가 새 정책에 맞지 않음

**해결**: 기존 사용자는 로그인은 가능하지만, 비밀번호 변경 시에만 새 정책 적용됨. 로그인 후 비밀번호 변경 유도.

### 문제 3: CSRF 토큰 오류

**원인**: Flask-WTF 미설치 또는 설정 오류

**해결**:
```bash
pip install Flask-WTF==1.2.1
# 서버 재시작
```
