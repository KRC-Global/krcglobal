# 글로벌사업관리시스템 서버 설치 가이드

> **환경**: 외부 인터넷 연결이 차단된 내부망 (Air-gapped Environment)  
> **대상 OS**: Windows Server 2019/2022 또는 Linux (CentOS/RHEL 7+, Ubuntu 20.04+)

---

## 2. 오프라인 설치 준비 (외부 PC에서)

### 2.1 Python Wheel 파일 다운로드

외부 인터넷이 되는 PC에서 wheel 파일을 미리 다운로드합니다.

```bash
# 프로젝트 루트의 requirements.txt 사용
pip download -r requirements.txt -d ./offline_packages/

# 또는 개별 패키지 다운로드
pip download Flask==2.3.3 Flask-SQLAlchemy==3.1.1 Flask-Cors==4.0.0 PyJWT==2.8.0 -d ./offline_packages/
```

### 2.2 Python 설치 파일 다운로드

| OS | 다운로드 URL | 파일명 |
|----|-------------|--------|
| Windows | https://www.python.org/ftp/python/3.11.7/ | python-3.11.7-amd64.exe |
| Linux (소스) | https://www.python.org/ftp/python/3.11.7/ | Python-3.11.7.tgz |

### 2.3 오프라인 패키지 폴더 구조

```
offline_install/
├── python-3.11.7-amd64.exe    # Windows용 Python 설치파일
├── Python-3.11.7.tgz          # Linux용 Python 소스
├── offline_packages/          # pip wheel 파일들
│   ├── Flask-2.3.3-py3-none-any.whl
│   ├── Flask_SQLAlchemy-3.1.1-py3-none-any.whl
│   ├── Flask_Cors-4.0.0-py3-none-any.whl
│   ├── PyJWT-2.8.0-py3-none-any.whl
│   └── ... (의존성 포함)
└── gbms_source/               # 어플리케이션 소스코드
```

## 3. 서버 설치 명령어

### 3.1 Windows Server

#### Step 1: Python 설치
```cmd
:: 관리자 권한으로 실행
python-3.11.7-amd64.exe /quiet InstallAllUsers=1 PrependPath=1

:: 설치 확인
python --version
pip --version
```

#### Step 2: 가상환경 생성
```cmd
cd C:\gbms
python -m venv venv
venv\Scripts\activate
```

#### Step 3: 오프라인 패키지 설치
```cmd
:: offline_packages 폴더에서 wheel 파일 설치
pip install --no-index --find-links=C:\offline_packages Flask Flask-SQLAlchemy Flask-Cors PyJWT waitress
```

#### Step 4: 데이터베이스 초기화
```cmd
cd C:\gbms\backend
python init_db.py
```

#### Step 5: 서버 실행 (개발/테스트)
```cmd
:: Backend (포트 5001)
cd C:\gbms\backend
python app.py

:: Frontend (포트 8000) - 새 CMD 창에서
cd C:\gbms
python -m http.server 8000
```

#### Step 6: 서버 실행 (운영 - Waitress)
```cmd
:: waitress를 사용한 운영 서버 실행
cd C:\gbms\backend
waitress-serve --port=5001 app:app
```

---

### 3.2 Linux (CentOS/RHEL/Ubuntu)

#### Step 1: Python 소스 컴파일 설치
```bash
# 의존성 설치 (사전에 RPM/DEB 패키지 준비 필요)
# CentOS/RHEL
sudo yum install -y gcc openssl-devel bzip2-devel libffi-devel zlib-devel

# Ubuntu
sudo apt install -y build-essential libssl-dev libffi-dev zlib1g-dev

# Python 소스 컴파일
tar -xzf Python-3.11.7.tgz
cd Python-3.11.7
./configure --enable-optimizations --prefix=/opt/python3.11
make -j$(nproc)
sudo make altinstall

# 심볼릭 링크 생성
sudo ln -sf /opt/python3.11/bin/python3.11 /usr/local/bin/python3
sudo ln -sf /opt/python3.11/bin/pip3.11 /usr/local/bin/pip3

# 설치 확인
python3 --version
pip3 --version
```

#### Step 2: 가상환경 생성
```bash
cd /opt/gbms
python3 -m venv venv
source venv/bin/activate
```

#### Step 3: 오프라인 패키지 설치
```bash
# offline_packages 폴더에서 wheel 파일 설치
pip install --no-index --find-links=/path/to/offline_packages Flask Flask-SQLAlchemy Flask-Cors PyJWT gunicorn
```

#### Step 4: 데이터베이스 초기화
```bash
cd /opt/gbms/backend
python init_db.py
```

#### Step 5: 서버 실행 (개발/테스트)
```bash
# Backend
cd /opt/gbms/backend
python app.py &

# Frontend
cd /opt/gbms
python -m http.server 8000 &
```

#### Step 6: 서버 실행 (운영 - Gunicorn)
```bash
# Gunicorn으로 Backend 실행
cd /opt/gbms/backend
gunicorn -w 4 -b 0.0.0.0:5001 app:app --daemon

# 또는 systemd 서비스로 등록 (아래 섹션 참고)
```

## 4. Systemd 서비스 등록 (Linux 운영 환경)

### 4.1 Backend 서비스 (`/etc/systemd/system/gbms-backend.service`)

```ini
[Unit]
Description=GBMS Backend API Server
After=network.target

[Service]
Type=simple
User=gbms
WorkingDirectory=/opt/gbms/backend
Environment="PATH=/opt/gbms/venv/bin"
ExecStart=/opt/gbms/venv/bin/gunicorn -w 4 -b 127.0.0.1:5001 app:app
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

### 4.2 Frontend 서비스 (`/etc/systemd/system/gbms-frontend.service`)

```ini
[Unit]
Description=GBMS Frontend Static Server
After=network.target

[Service]
Type=simple
User=gbms
WorkingDirectory=/opt/gbms
Environment="PATH=/opt/gbms/venv/bin"
ExecStart=/opt/gbms/venv/bin/python -m http.server 8000
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

### 4.3 서비스 등록 및 시작
```bash
sudo systemctl daemon-reload
sudo systemctl enable gbms-backend gbms-frontend
sudo systemctl start gbms-backend gbms-frontend

# 상태 확인
sudo systemctl status gbms-backend
sudo systemctl status gbms-frontend
```

## 5. Nginx 리버스 프록시 설정 (권장)

### 5.1 Nginx 설정 (`/etc/nginx/conf.d/gbms.conf`)

```nginx
server {
    listen 80;
    server_name gbms.local;  # 내부 도메인 또는 IP

    # Frontend (정적 파일)
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # Backend API
    location /api/ {
        proxy_pass http://127.0.0.1:5001/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

### 5.2 Nginx 적용
```bash
sudo nginx -t
sudo systemctl reload nginx
```

## 6. 방화벽 설정

### Windows Server
```cmd
:: 포트 개방
netsh advfirewall firewall add rule name="GBMS Frontend" dir=in action=allow protocol=tcp localport=8000
netsh advfirewall firewall add rule name="GBMS Backend" dir=in action=allow protocol=tcp localport=5001
```

### Linux (firewalld)
```bash
sudo firewall-cmd --permanent --add-port=8000/tcp
sudo firewall-cmd --permanent --add-port=5001/tcp
sudo firewall-cmd --reload
```

### Linux (ufw - Ubuntu)
```bash
sudo ufw allow 8000/tcp
sudo ufw allow 5001/tcp
```

## 8. 문제 해결

### 8.1 "pip: command not found"
```bash
# Python Scripts 경로를 PATH에 추가
# Windows
set PATH=%PATH%;C:\Python311\Scripts

# Linux
export PATH=$PATH:/opt/python3.11/bin
```

### 8.2 "ModuleNotFoundError: No module named 'flask'"
```bash
# 가상환경 활성화 확인
# Windows
venv\Scripts\activate

# Linux
source venv/bin/activate

# 패키지 재설치
pip install --no-index --find-links=./offline_packages -r requirements.txt
```

### 8.3 SQLite 버전 문제
```bash
# Python에 내장된 SQLite 버전 확인
python -c "import sqlite3; print(sqlite3.sqlite_version)"
# 3.35.0 이상 필요
```
