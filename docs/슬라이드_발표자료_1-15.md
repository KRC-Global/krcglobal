## 슬라이드 3: 시스템 개요
**제목**: 시스템 개요

**주요 내용**:
- **시스템명**: 글로벌사업관리시스템 (GBMS)
- **목적**: 글로벌사업처 업무 디지털화 및 통합 관리
- **적용 범위**:
  - 해외기술용역사업
  - 국제협력사업 (ODA)
  - 메탄감축사업
  - 해외진출지원사업
  - 해외사무소 관리

**핵심 수치**:
- 동시 사용자: ~100명
- 관리 사업 수: 200+ 프로젝트
- 협력 국가: 40개국 이상
- 해외사무소: 8개소

**비주얼 요소**:
- 중앙: 시스템 구성도 (원형 다이어그램)
- 주변: 핵심 수치 카드

## 슬라이드 4: 추진 배경
**제목**: Why GBMS?

**문제점 (AS-IS)**:
1. **분산된 데이터 관리**
   - Excel, 문서 파일 산재
   - 실시간 현황 파악 어려움

2. **비효율적 업무 프로세스**
   - 수작업 집계 및 보고
   - 중복 입력 발생

3. **정보 공유 제한**
   - 부서 간 협업 장벽
   - 의사결정 지연

4. **체계적 관리 미흡**
   - 사업별 진행 상황 추적 어려움
   - 예산 집행 가시성 부족

**비주얼 요소**:
- 좌측: AS-IS (문제 상황 일러스트)
- 화살표: 전환
- 우측: TO-BE (시스템 화면 프리뷰)

## 슬라이드 6: 시스템 아키텍처
**제목**: System Architecture

**기술 스택**:

**Frontend**:
- HTML5, CSS3, JavaScript (Vanilla)
- KRDS Design System
- Leaflet (GIS)

**Backend**:
- Python Flask 2.3.3
- SQLAlchemy 3.1.1
- JWT Authentication

**Database**:
- SQLite with WAL Mode
- 100명 동시 접속 최적화

**Deployment**:
- 내부망 환경
- Windows Server
- 포트: 5001 (Backend), 8000 (Frontend)

**비주얼 요소**:
- 3계층 아키텍처 다이어그램
- Frontend / Backend / Database 레이어 표시

## 슬라이드 15: 보안 및 권한 관리
**제목**: 안전한 데이터 관리

**보안 체계**:

**인증 (Authentication)**:
- JWT 기반 토큰 인증
- 8시간 세션 유지
- 자동 로그아웃

**권한 관리 (Authorization)**:
- 역할 기반 접근 제어 (RBAC)
  - 관리자 (Admin)
  - 매니저 (Manager)
  - 사용자 (User)
- 부서별 데이터 접근 제한
  - 글로벌농업개발부
  - 글로벌사업부
  - 농식품국제개발협력센터

**데이터 보호**:
- 내부망 환경 운영
- 접근 로그 기록
- 민감 정보 암호화

**감사 추적**:
- 모든 활동 로그 저장
- IP 주소 기록
- 사용자 행동 추적

**비주얼 요소**:
- 권한 레벨 피라미드
- 보안 프로세스 플로우차트
