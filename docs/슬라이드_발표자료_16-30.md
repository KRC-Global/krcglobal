## 슬라이드 24: API 구조
**제목**: RESTful API 설계

**API 엔드포인트**:

**인증 API** (`/api/auth`):
- POST /login - 로그인
- POST /logout - 로그아웃
- GET /validate - 토큰 검증

**사업 관리 API** (`/api/projects`):
- GET /projects - 목록 조회
- POST /projects - 등록
- PUT /projects/:id - 수정
- DELETE /projects/:id - 삭제

**문서 API** (`/api/documents`):
- POST /upload - 업로드
- GET /download/:id - 다운로드

**통계 API** (`/api/dashboard`):
- GET /stats - 통계 조회
- GET /charts - 차트 데이터

**API 특징**:
- JSON 기반 통신
- JWT 토큰 인증
- 에러 핸들링
- 페이지네이션

**비주얼 요소**:
- API 구조도
- 요청/응답 예시
